## <a name="limitations"></a><span data-ttu-id="16c29-155">制限事項</span><span class="sxs-lookup"><span data-stu-id="16c29-155">Limitations</span></span>

<span data-ttu-id="16c29-156">バック プレーンを使用して、メッセージの最大スループットは、クライアントは、1 台のサーバー ノードに直接話すときよりも低い。</span><span class="sxs-lookup"><span data-stu-id="16c29-156">Using a backplane, the maximum message throughput is lower than it is when clients talk directly to a single server node.</span></span> <span data-ttu-id="16c29-157">バック プレーンがバック プレーンがボトルネックになることができますので、すべてのノードにすべてのメッセージを転送するためです。</span><span class="sxs-lookup"><span data-stu-id="16c29-157">That's because the backplane forwards every message to every node, so the backplane can become a bottleneck.</span></span> <span data-ttu-id="16c29-158">この制限は、問題であるかどうかは、アプリケーションによって異なります。</span><span class="sxs-lookup"><span data-stu-id="16c29-158">Whether this limitation is a problem depends on the application.</span></span> <span data-ttu-id="16c29-159">たとえば、SignalR の一般的なシナリオを示します。</span><span class="sxs-lookup"><span data-stu-id="16c29-159">For example, here are some typical SignalR scenarios:</span></span>
