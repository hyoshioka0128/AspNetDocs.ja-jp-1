### <a name="page-directive"></a>@Page ディレクティブ

ASP.NET Web フォームには、通常、ページのプロパティと構成情報を指定できるようにするディレクティブが含まれています。 ディレクティブは、ページの処理方法の手順として ASP.NET によって使用されますが、ブラウザーに送信されるマークアップの一部としてはレンダリングされません。

最もよく使用されるディレクティブは `@Page` ディレクティブです。このディレクティブを使用すると、次のようなページに対して多くの構成オプションを指定できます。

1. などC#、ページ内のコードのサーバープログラミング言語。
2. ページが、ページに直接配置されているページであるかどうか (単一ファイルページと呼ばれます)、または分離コードページと呼ばれる別のクラスファイル内のコードを含むページであるかどうか。
3. ページに関連付けられたマスターページがあるかどうか。したがって、コンテンツページとして処理される必要があります。
4. デバッグオプションとトレースオプション。

ページに `@Page` ディレクティブが含まれていない場合、またはディレクティブに特定の設定が含まれていない場合は、 *web.config 構成ファイル*または machine.config 構成ファイルから設定が継承さ*れます。* Machine.config*ファイルは*、コンピューター上で実行されているすべてのアプリケーションに対して追加の構成設定を提供します。

> [!NOTE] 
> 
> *Machine.config*は、可能なすべての構成設定についての詳細も提供します。

### <a name="web-server-controls"></a>Web サーバーコントロール

ほとんどの ASP.NET Web フォームアプリケーションでは、ボタン、テキストボックス、リストなどのページとの対話をユーザーに許可するコントロールを追加します。 これらの Web サーバーコントロールは、HTML ボタンや入力要素に似ています。 ただし、これらはサーバーで処理されるので、サーバーコードを使用してプロパティを設定できます。 また、これらのコントロールは、サーバーコードで処理できるイベントを発生させます。

サーバーコントロールは、ページの実行時に ASP.NET が認識する特別な構文を使用します。 ASP.NET サーバーコントロールのタグ名は `asp:` プレフィックスで始まります。 これにより、ASP.NET はこれらのサーバーコントロールを認識して処理できます。 コントロールが .NET Framework に含まれていない場合は、プレフィックスが異なる場合があります。 ASP.NET サーバーコントロールには、`asp:` プレフィックスに加えて、`runat="server"` 属性と、サーバーコード内のコントロールを参照するために使用できる `ID` も含まれています。

ページが実行されると、ASP.NET はサーバーコントロールを識別し、それらのコントロールに関連付けられているコードを実行します。 多くのコントロールは、ブラウザーに表示されるときに HTML またはその他のマークアップをページに表示します。

### <a name="server-code"></a>サーバーコード

ほとんどの ASP.NET Web フォームアプリケーションには、ページの処理時にサーバー上で実行されるコードが含まれています。 前述のように、サーバーコードを使用して、ListView コントロールにデータを追加するなど、さまざまな操作を実行できます。 ASP.NET は、 C#、Visual Basic、J# など、サーバーで実行する多くの言語をサポートしています。

ASP.NET は、Web ページのサーバーコードを記述する2つのモデルをサポートしています。 単一ファイルモデルでは、ページのコードは script 要素内にあります。この要素には、開始タグに `runat="server"` 属性が含まれています。 別の方法として、分離コードモデルと呼ばれる別のクラスファイルにページのコードを作成することもできます。 この場合、ASP.NET Web フォームページには通常、サーバーコードが含まれていません。 代わりに、`@Page` ディレクティブには、 *.aspx*ページとそれに関連付けられた分離コードファイルをリンクする情報が含まれています。

`@Page` ディレクティブに含まれる `CodeBehind` 属性は、個別のクラスファイルの名前を指定し、`Inherits` 属性は、そのページに対応する分離コードファイル内のクラスの名前を指定します。

### <a name="master-pages"></a>マスター ページ

マスターページは、マスター (たとえば、ASP.NET) という拡張子を持つファイルで、静的なテキスト、HTML 要素、およびサーバーコントロールを含めることができる定義済みのレイアウトが含まれて*います。* マスターページは、通常の *.aspx*ページで使用される `@Page` ディレクティブを置き換える特殊な `@Master` ディレクティブによって識別されます。

マスターページには、`@Master` ディレクティブに加えて、ページの最上位レベルの HTML 要素 (`html`、`head`、`form`など) もすべて含まれています。 たとえば、上で追加したマスターページで、レイアウトに HTML `table` を使用し、会社のロゴ、静的テキスト、およびサーバーコントロールの `img` 要素を使用して、サイトの共通のメンバーシップを処理します。 マスターページの一部として、任意の HTML および任意の ASP.NET 要素を使用できます。

すべてのページに表示される静的テキストとコントロールに加えて、マスターページには1つ以上の**ContentPlaceHolder**コントロールも含まれています。 これらのプレースホルダーコントロールは、置き換え可能なコンテンツが表示される領域を定義します。 さらに、置き換え可能なコンテンツは、**コンテンツ**サーバーコントロールを使用して*default.aspx*などのコンテンツページで定義されます。
