### <a name="pessimistic-concurrency-locking"></a><span data-ttu-id="13285-123">ペシミスティック同時実行 (ロック)</span><span class="sxs-lookup"><span data-stu-id="13285-123">Pessimistic Concurrency (Locking)</span></span>

<span data-ttu-id="13285-129">ロックの利用には短所があります。</span><span class="sxs-lookup"><span data-stu-id="13285-129">Managing locks has disadvantages.</span></span> <span data-ttu-id="13285-130">プログラムが複雑になります。</span><span class="sxs-lookup"><span data-stu-id="13285-130">It can be complex to program.</span></span> <span data-ttu-id="13285-131">相当なデータベース管理リソースが必要になります。アプリケーションの利用者数が増えると、パフォーマンス上の問題を引き起こすことがあります。</span><span class="sxs-lookup"><span data-stu-id="13285-131">It requires significant database management resources, and it can cause performance problems as the number of users of an application increases.</span></span> <span data-ttu-id="13285-132">そのような理由から、一部のデータベース管理システムはペシミスティック コンカレンシーに対応していません。</span><span class="sxs-lookup"><span data-stu-id="13285-132">For these reasons, not all database management systems support pessimistic concurrency.</span></span> <span data-ttu-id="13285-133">Entity Framework には、組み込みのサポートはありません。このチュートリアルでは、実装方法については説明しません。</span><span class="sxs-lookup"><span data-stu-id="13285-133">The Entity Framework provides no built-in support for it, and this tutorial doesn't show you how to implement it.</span></span>

### <a name="detecting-concurrency-conflicts"></a><span data-ttu-id="13285-158">同時実行の競合の検出</span><span class="sxs-lookup"><span data-stu-id="13285-158">Detecting Concurrency Conflicts</span></span>
